## Product Brief: AI in K–12 Resource Navigator

**Platform:** Replit (lightweight web app)
**Goal:** Build a continuously curated, practitioner-facing “AI resource wiki” that helps users quickly find trusted, current, and role-relevant resources on AI in K–12 education and equity.

---

## 1. Problem to Solve

Users (SPOs, grantees, educators) are facing:

* Too many AI resources, unclear quality
* Outdated materials still circulating
* No clear “starting point” based on role or knowledge level
* Lack of trusted voices and curated pathways

This tool should **reduce noise and guide decisions**, not just aggregate content.

---

## 2. Core Product Concept

A **simple, fast, searchable web app** that:

* Curates high-quality AI resources (articles, videos, reports, tools)
* Organizes them by **audience, topic, and level**
* Adds **human-readable annotations**
* Flags **freshness and relevance**
* Evolves over time like a **lightweight public wiki**

---

## 3. Key User Flows

### Flow 1: “I’m new—where do I start?”

User clicks:
→ “Start Here”
→ Sees 5–10 essential resources with simple explanations

---

### Flow 2: “I need resources for my partners”

User filters by:
→ Audience: “K–12 practitioners”
→ Topic: “AI & Equity”
→ Gets curated list with summaries

---

### Flow 3: “What’s current vs outdated?”

User sees:
→ Tags: “Foundational”, “Current”, “Needs update”
→ Optional “What’s changing now” section

---

### Flow 4: “Who should I trust?”

User opens:
→ “Trusted Voices”
→ Sees organizations (e.g., National Center on Education and the Economy)
→ Short explanation of why they matter

---

## 4. MVP Feature Set (Build This First)

### A. Resource Database (core schema)

Each resource = one record:

```
{
  id: string,
  title: string,
  url: string,
  source: string,
  format: "article" | "video" | "podcast" | "report" | "tool",
  audience: ["practitioner", "grantee", "leader", "internal"],
  topic: ["AI in K12", "Equity", "Classroom", "Policy", "Market"],
  level: "beginner" | "intermediate" | "advanced",
  summary: string,
  why_useful: string,
  watch_out: string,
  freshness: "foundational" | "current" | "needs_update",
  date_added: timestamp
}
```

---

### B. Core Pages

* Title: “AI in K–12 Resource Navigator”

## 5. UI/UX Guidelines

Keep it:

* Clean, minimal, fast
* Card-based layout
* No heavy dashboards

Visual elements:

* Color-coded freshness tags:

  * Green = Current
  * Gray = Foundational
  * Yellow = Needs update

* Tag pills for:

  * Audience
  * Topic
  * Level

## 10. Success Criteria (MVP)

The tool works if:

* A user can find **3–5 relevant, high-quality resources in <2 minutes**
* They understand **why each resource matters**
* They can distinguish **current vs outdated thinking**
* They can **confidently share with partners**
